# Project resources

Use these locations to navigate the `elianacorre.com` project. Link to canonical artifacts instead of copying their contents between systems.

| Resource | Location | Purpose |
| --- | --- | --- |
| Linear | [elianacorre.com project](https://linear.app/niama/project/elianacorrecom-9ad653bd32f2) | Actionable work, priorities, ownership, dependencies, acceptance criteria, and status |
| GitHub | [niama-fr/elianacorre.com](https://github.com/niama-fr/elianacorre.com) | Code, branches, pull requests, CI, releases, and deployment provenance |
| Convex | Projects `eliana-corre:elianacorre-com-staging` and `eliana-corre:elianacorre-com-b1869` | Isolated staging and production deployments, environment variables, and data |
| Google Drive | [elianacorre.com folder](https://drive.google.com/drive/folders/1Cb9B6AijJrLzzli6bkQkFR6fhc0buaZA) in the Niama account | Original assets and collaborative business documents |
| Processing register | [Registre des activités de traitement](https://docs.google.com/document/d/1pXfz8B3GrsWod0GYjYmGkPxPlxD9JGt0bOFRXTpP0CU/edit) | Internal GDPR processing register and provider/transfer verification |
| Obsidian | Niama vault → `Elianacorre.com` | Research, exploration, meeting notes, and evolving project knowledge |
| Repository ADRs | [`docs/adr/`](../adr/) and context-specific `docs/adr/` directories | Finalized technical and architectural decisions |

The Drive and Obsidian locations are described by account, vault, and artifact name so this repository does not depend on a contributor's local filesystem path.

Canonical session syntheses live in the Niama Obsidian vault's `Elianacorre.com` project area. They should summarize current state, open loops, canonical links, and the consumption/manual-responsibility report instead of duplicating full Linear issues, pull request descriptions, or repository diffs.
